出入库记录数据结构示例：

{
  "_id": "系统自动生成的ID",
  "productId": "产品ID(可选)",
  "productName": "产品名称",
  "quantity": 10,
  "type": "out",  // 记录类型: "in" 表示入库, "out" 表示出库
  "createTime": "2025-04-12 22:02:00",  // 创建时间
  "orderId": "订单ID(可选)",
  "orderNote": "预存产品扣减: ORD20230001",  // 操作说明
  "customerInfo": "客户名称或ID(可选)"  // 客户信息
}

需要注意的事项：
1. 确保records集合已创建并有权限写入
2. 出库和入库记录都需要设置type字段
3. quantity字段必须是数字类型
4. createTime应该使用服务器时间(db.serverDate())
5. 所有订单处理、预存扣除等操作都应该创建对应的出入库记录

特殊情况：
- 对于预存产品扣减，需要添加orderNote说明来源
- 手动扣减时，应该添加足够的信息标识来源
- 出库记录应该包含充分信息以便用户识别用途
